About

Accessibility Bug Reports Labeler is a Jira plugin that utilizes a language model to automatically label issue reports as either accessibility-related or not. The plugin was developed by fine-tuning Google's pre-trained ALBERT model on 7,096 manually labeled issues, achieving 95% F1-score, recall, and precision during testing. When installed in Jira, the plugin analyzes the title and description of new issues and applies a label indicating if it is an accessibility report or not. This saves time for developers by automatically categorizing incoming issues and helping prioritize accessibility defects. The high accuracy of the model results from training it on a substantially labeled dataset. The plugin consists of three parts: the classification model, APIs, and Jira Plugin.

Set up

The following are the steps to set up the plugin.

  • Step 1

    Copy the plugin installation link
    https://accessibility-labeler-plugin.onrender.com/install

  • Step 2

    Sign in to your Jira account and navigate to the "Manage Apps" setting found under "Apps" at the top of your dashboard

    Note: Be sure to enable development mode for your Jira instance by toggling the option under Manage Apps > Settings > Enable Development Mode

  • Step 3

    - Click on "Upload App" and paste the URL into it.
    - Click on "Upload App" and paste the URL into it and click "Upload"

Issue Labeler Model

The plugin’s prediction engine is an accessibility-classification deep-learning language model implemented by fine-tuning Google’s pre-trained ALBERT language model. Introduced by Google in 2019, ALBERT, which stands for A Lite BERT, is a smaller version of the original BERT language model. ALBERT achieves a smaller size due to the factorization of BERT’s embedding parameters leading to an “80% reduction in the parameters of the projection block, at the expense of only a minor drop in performance” (https://ai.googleblog.com/2019/12/albert-lite-bert-for-self-supervised.html)

Issue Labeler Jira Plugin

This Jira plugin is an extension of the accessibility labeler model. It is installable into Jira Cloud instances and is viewable in an issue's page. The plugin will passively analyze an issue's title and description by communicating with the issue labeler model and display prediction types ranked by their confidence score.
